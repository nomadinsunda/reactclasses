
지금부터는 **현재 프로젝트를 `@tailwindcss/vite` 방식으로 완전히 정리하는 절차** 를, **React + Vite + Tailwind v4 + daisyUI 사용 가능성**까지 포함해서 정확하게 설명드리겠습니다.

Tailwind 공식 문서는 Vite에서는 `@tailwindcss/vite` 플러그인 방식이 가장 매끄러운 통합 방법이라고 안내하고 있고, Tailwind v4 블로그도 **PostCSS보다 Vite 플러그인 쪽이 더 좋은 성능**을 준다고 설명합니다. 또한 daisyUI 공식 문서도 Vite 환경에서 `@tailwindcss/vite`와 `@plugin "daisyui";` 구성을 안내합니다. ([Tailwind CSS][1])

# 최종 목표

정리 후 구조는 대략 이렇게 됩니다.

* `postcss.config.js` 삭제
* `@tailwindcss/postcss` 제거
* `postcss` 제거
* `vite.config.js`에 `@tailwindcss/vite` 등록
* `src/index.css`는 `@import "tailwindcss";` 사용
* daisyUI를 쓴다면 CSS 파일 안에서 `@plugin "daisyui";` 사용

Tailwind v4는 CSS 중심 구성을 사용하고, daisyUI 5도 Tailwind 4에서 CSS 파일 안에 plugin을 선언하는 방식을 지원합니다. ([Tailwind CSS][1])

---

# 1) 먼저 현재 상태에서 제거할 것

업그레이드 도구가 이미 `@tailwindcss/postcss` 기반으로 바꿔놓은 상태라면, Vite 방식으로 갈아탈 때는 아래를 정리하시면 됩니다.

터미널에서:

```bash
npm remove @tailwindcss/postcss postcss
```

`autoprefixer`는 이미 업그레이드 로그에서 제거된 상태였으므로 보통 추가 조치는 필요 없습니다. Tailwind v4의 PostCSS 설치 문서도 핵심 패키지를 `@tailwindcss/postcss`로 설명하고 있고, Vite 플러그인 방식에서는 그 패키지가 필요 없습니다. ([Tailwind CSS][1])

---

# 2) Vite 플러그인 설치

```bash
npm install -D @tailwindcss/vite
```

Tailwind 공식 Vite 설치 문서는 `@tailwindcss/vite`를 설치하고 Vite 설정에 등록하는 흐름을 제시합니다. daisyUI의 Vite 설치 문서도 동일한 방향입니다. ([Tailwind CSS][1])

---

# 3) `vite.config.js` 수정

기존 `vite.config.js`가 React 기준으로 대략 이런 식일 가능성이 큽니다.

```js
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
});
```

이걸 이렇게 바꾸시면 됩니다.

```js
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  plugins: [react(), tailwindcss()],
});
```

이 구성이 Tailwind 공식 Vite 방식의 핵심입니다. ([Tailwind CSS][1])

---

# 4) `postcss.config.js` 삭제

이제 Tailwind를 PostCSS 설정 파일로 연결하지 않으므로 이 파일은 제거합니다.

삭제 대상:

```bash
postcss.config.js
```

또는 프로젝트에 따라:

```bash
postcss.config.mjs
postcss.config.cjs
```

중 하나일 수 있습니다.

Vite 플러그인 방식에서는 Tailwind 연결이 `vite.config.*`로 이동하므로, 이 파일이 남아 있으면 오히려 구조가 중복되어 헷갈릴 수 있습니다. 공식 Vite 설치 가이드도 PostCSS 파일을 전제로 하지 않고 Vite plugin 구성을 안내합니다. ([Tailwind CSS][1])

---

# 5) `src/index.css` 정리

Tailwind v4의 기본 진입 방식은 `@import "tailwindcss";`입니다. ([Tailwind CSS][1])

## daisyUI를 쓰지 않는 경우

```css
@import "tailwindcss";
```

## daisyUI를 쓰는 경우

```css
@import "tailwindcss";
@plugin "daisyui";
```

이건 daisyUI 공식 설치 문서의 v5 + Tailwind 4 방식입니다. daisyUI 5 릴리즈 노트도 Tailwind 4에서 CSS 안에 plugin을 선언하는 방식을 안내합니다. ([daisyui.com][2])

---

# 6) `tailwind.config.js`는 어떻게 할까?

여기서 많이 헷갈립니다.

Tailwind v4에서는 많은 설정이 CSS 중심으로 이동했기 때문에, **기존 `tailwind.config.js`가 반드시 필요하다고 볼 수는 없습니다.** 하지만 프로젝트에 다음 같은 커스텀이 들어 있다면 아직 남겨둘 이유가 있습니다. ([Tailwind CSS][3])

예를 들면:

* 커스텀 theme 설정
* custom colors
* fontFamily
* spacing 확장
* animation, keyframes
* daisyUI theme 옵션

즉 정리 기준은 이렇습니다.

## 삭제해도 되는 경우

`tailwind.config.js`가 거의 비어 있거나, 예전 `content`, `plugins` 정도만 있었던 경우

## 남겨야 하는 경우

실제 커스텀 값이 들어 있는 경우

다만 **daisyUI plugin 등록은 v4에서는 CSS의 `@plugin "daisyui";` 쪽으로 옮기는 것이 더 자연스럽습니다.** daisyUI 공식 문서가 그렇게 안내합니다. ([daisyui.com][2])

# 7) `main.jsx` 또는 `main.tsx` 확인

이 파일에서는 CSS import만 유지되면 됩니다.

```js
import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App";
import "./index.css";

ReactDOM.createRoot(document.getElementById("root")).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

Tailwind 공식 Vite 가이드도 CSS 파일을 애플리케이션 진입점에서 import하는 흐름을 전제로 합니다. ([Tailwind CSS][1])

# 8) 현재 프로젝트 기준 최종 추천 형태

## `package.json`의 devDependencies 예시

daisyUI를 쓴다면 대략 이런 핵심 조합이면 됩니다.

```json
{
  "devDependencies": {
    "@tailwindcss/vite": "^4.2.2",
    "daisyui": "^5.5.14",
    "tailwindcss": "^4.2.2"
  }
}
```

버전은 실제 프로젝트에 맞게 맞추시면 됩니다. 업그레이드 로그 기준으로 Tailwind는 4.2.2, daisyUI는 5.5.14가 감지되었습니다. Tailwind v4와 daisyUI 5는 호환됩니다. ([daisyui.com][4])

# 10) 자주 실수하는 부분

## 1. `postcss.config.js`를 안 지우고 놔두는 경우

반드시 문제를 일으키는 건 아니지만, 구조가 이중화되어 혼란스러워집니다. Vite 플러그인으로 완전히 정리하려면 제거하는 편이 깔끔합니다. ([Tailwind CSS][1])

## 2. `src/index.css`에 옛날 문법이 남아 있는 경우

예전 v3 문법인:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

대신, v4에서는 기본적으로:

```css
@import "tailwindcss";
```

를 사용합니다. ([Tailwind CSS][3])

## 3. daisyUI를 아직 `tailwind.config.js`의 `plugins`에만 남겨두는 경우

Tailwind 4 + daisyUI 5에서는 CSS에서 `@plugin "daisyui";` 방식이 더 직접적입니다. ([daisyui.com][2])

## 4. Sass와 섞는 경우

Tailwind 공식 호환성 문서는 v4가 Sass, Less, Stylus 같은 전처리기와 함께 쓰는 방향을 권장하지 않는다고 설명합니다. `.scss` 중심이면 추가 점검이 필요합니다. ([Tailwind CSS][5])
